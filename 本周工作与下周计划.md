# 本周工作与下周计划

本文档用于简要说明：本组本周已经完成了哪些工作、为什么这些工作有价值，以及下周准备继续推进哪些内容。

---

## 一、本周完成了什么

### 1. 调整了 API 调用方式，去掉硬编码依赖

本周我们对原有 API 调用方式做了整理和修改，把原先偏硬编码的调用模式，逐步调整为更贴近课程统一契约的方式。

主要变化包括：

- 将 LLM 调用入口改为统一网关思路
- 将原先分散、偏本地脚本式的调用方式，改为更适合后续联调的结构
- 为后续 `HTTP + Redis + consultation service` 的统一接入做准备

这项工作的价值在于：

- 降低后续联调时的接口改动成本
- 避免把模型调用逻辑写死在单一脚本或单一页面流程里
- 为后续适配课程契约提供基础

### 2. 梳理并重构了本组的协作思路

本周我们重点调研并梳理了本组在全院统一架构中的更合适位置，重新思考了：

- 本组应该承担什么角色
- 本组与其他小组之间如何协作
- `HTTP`、`Redis`、内部 MDT 引擎之间的信息流应该怎样设计
- 二次补充信息、有限轮次重评和最终方案收束应如何实现

在这个过程中，我们也识别出了一些当前统一规划中对本组来说仍然缺少的中间模块，例如：

- consultation 输入输出的标准化层
- 补充信息的协同整合层
- 二次补充后的重评与最终收束控制层

这项工作的价值在于：

- 让本组后续开发不再只是“继续堆功能”
- 而是先把系统位置、协作边界和信息循环想清楚
- 避免后面反复返工

### 3. 形成了较完整的适配说明文档

本周我们已经把本组如何接入统一契约的思路整理成了单独文档，包括：

- 为什么本组在合并时会遇到结构性困难
- 本组更适合的嵌入方式
- `HTTP` 输入输出建议
- `Redis` 事件回流建议
- 二次补充与最终收束逻辑

这项工作有两个直接作用：

- 便于和老师、同学沟通本组思路
- 便于下周直接按文档分工落实

---

## 二、本周工作的意义

如果用一句话总结，本周的工作重点不是继续扩展演示页面，而是：

**先把本组从“能跑的原型”逐步整理为“能接入统一系统的能力服务”。**

这样做的意义在于：

- 有助于后续和其他小组对接
- 有助于提高本组模型能力在全院系统中的可用性
- 有助于把本组原有的多专科综合判断与方案收束优势真正发挥出来

---

## 三、下周准备做什么

### 1. 进一步确认协作架构和流程，并开始落实

下周我们会继续和统一架构对齐，进一步确认：

- consultation 服务的接口形式
- 补充信息如何进入系统
- `HTTP` 与 `Redis` 的职责边界
- 二次补充、重评和最终收束的具体流程

在确认思路后，会开始把这些设计逐步落实到代码中。

### 2. 编写测试案例，重点覆盖边缘情况

下周我们会开始补测试，重点关注：

- 边缘情况
- 鲁棒性
- 稳定性
- 模型输出质量

例如可能重点关注：

- 输入信息缺失或格式不完整时的表现
- 多轮补充后是否能够稳定收束
- 不同专科意见冲突时的处理
- Redis 事件回流后的重评是否正确触发

这部分工作很重要，因为本组系统后续如果要真正参与联调，稳定性会比单次演示更重要。

### 3. 优化模型运行时间

下周我们还会继续关注模型运行效率，包括：

- 哪些步骤需要完整重评
- 哪些步骤可以只做轻量更新
- 多轮补充时如何减少重复计算

这部分优化的目标是：

- 缩短 consultation 响应时间
- 提高多轮流程的可用性
- 为后续联调和演示降低等待成本

### 4. 继续优化数据库与存储设计

下周我们准备进一步学习其他小组在数据库和数据管理上的做法，并结合本组需求继续优化：

- consultation 结果存储
- 补充信息存储
- 状态快照管理
- 多轮评估历史记录

这项工作有助于提升：

- 系统可维护性
- 多轮流程的一致性
- 后续联调和调试效率

---

## 四、下周工作目标

下周的总体目标可以概括为：

1. 把协作架构从“说明文档”推进到“初步实现”
2. 把系统从“单次演示能力”推进到“可联调能力”
3. 把系统从“能出结果”推进到“结果更稳定、更快、更可复用”

---

## 五、简要总结

本周我们主要完成了两类基础工作：

- 对模型调用和接口接入方式做了面向契约的调整
- 对本组在统一系统中的协作关系、信息循环和缺失模块做了较系统的梳理

下周我们会把重点放在：

- 架构确认与落地
- 测试与鲁棒性
- 运行效率优化
- 数据管理与存储优化

整体上，我们希望逐步把本组项目从一个展示型原型，推进为一个更稳定、更容易和其他小组协作的 MDT/术前决策能力服务。
