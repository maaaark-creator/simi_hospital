# 🏥 智能术前评估与手术决策多Agent系统

**Intelligent Multi-Agent System for Preoperative Evaluation & Surgical Decision Making**

---

## 📌 项目简介

本项目旨在构建一个**贴近真实临床流程的术前评估与手术决策系统**，通过多智能体（Multi-Agent）协作模拟医院中不同角色（外科、麻醉、MDT、患者等）的决策过程，实现：

* 多学科并行评估（Parallel Evaluation）
* 动态病情监测（Dynamic Monitoring）
* 医患共同决策（Shared Decision Making）
* 基于权威数据库的辅助决策（Evidence-Based Decision）

系统不仅关注医学合理性，还综合考虑患者的**经济、心理、家庭等现实因素**，最终输出更加真实可信的手术决策。

## 🤖 Agent 设计

### 1️⃣ 麻醉 Agent（已实现 ✅）

**职责：**

* 判断是否具备麻醉与手术指征
* 提供麻醉方式与剂量建议
* 评估围术期风险（如 ASA 分级）

**特点：**

* 基于**权威医学数据库**进行检索增强（RAG）
* 实现“输入 → 知识对齐 → 输出建议”的闭环

---

### 2️⃣ 外科 Agent

**职责：**

* 判断是否需要手术
* 提出多种手术方案（保守 vs 激进）
* 评估预后与风险

---

### 3️⃣ MDT Agent（多学科会诊）

**职责：**

* 综合多个专科意见（影像、肿瘤、内科等）
* 给出整体治疗策略（非单一手术视角）

---

### 4️⃣ 患者 Agent（关键创新点 ⭐）

**职责：**

* 模拟患者真实决策行为
* 综合以下因素：

  * 💰 经济条件
  * 🧠 心理状态
  * 👨‍👩‍👧 家庭支持
  * 📉 风险接受程度

👉 使系统从“医学最优”升级为“现实最优”

---

### 5️⃣ Headmaster Agent（决策核心）

**职责：**

* 汇总所有 Agent 输出
* 权衡冲突信息
* 输出最终推荐手术方案

👉 类似真实医院中的“主任医师 / 主刀决策者”

---

### 6️⃣ 状态检测模块（Dynamic Monitoring）

**职责：**

* 持续监测患者状态变化（生命体征、病情进展）
* 判断是否需要重新评估

👉 实现真实临床中的“动态决策”

---

## 🔄 决策闭环机制

系统不是一次性决策，而是**循环决策系统**：

1. 医生提出方案
2. 患者评估接受度
3. 若拒绝 → 外科重新设计方案
4. 若病情变化 → 全流程重新评估

👉 模拟真实医院中的“反复沟通与调整”

---

## 🗄️ 数据来源（关键优势 ⭐）

本项目使用**真实权威医学资料构建数据库**：

* 临床指南（Guidelines）
* 麻醉规范（Anesthesia Protocols）
* 手术适应症标准
* 围术期管理文献

**特点：**

* 可结构化（适配 Agent）
* 可检索（RAG）
* 可扩展（支持多疾病）
