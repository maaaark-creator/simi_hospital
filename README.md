# 🏥 MDT驱动的智能术前评估与决策多Agent系统

**MDT-Driven Multi-Agent System for Clinical Decision Making**

---

## 📌 项目简介（更新版）

本项目构建一个以 **MDT（多学科会诊）为核心调度机制** 的智能医疗决策系统，用于模拟真实医院中的术前评估与治疗决策流程。

与传统“多模型并行输出”不同，本系统：

👉 由 **MDT Agent 负责决策调度（Decision Orchestrator）**
👉 按需唤醒不同专科 Agent（On-demand Activation）
👉 实现更真实、更高效的临床决策路径

系统最终不仅输出手术方案，还提供：

* 🧠 预后评估（Prognosis）
* 📋 综合治疗建议（非手术路径）
* 👨‍⚕️ 主治医生视角决策（Attending-level reasoning）

## 🤖 Agent 体系（升级版）

---

### ⭐ 1️⃣ MDT 调度 Agent（核心创新）

**角色定位：**

> 类似真实医院中的“会诊发起 + 分诊系统”

**职责：**

* 解析病例复杂度
* 判断涉及的专科范围
* **动态唤醒对应 Agent**
* 统一收集多学科意见

**调用策略：**

| Agent类型   | 调用方式   |
| --------- | ------ |
| 麻醉        | ✅ 必须调用 |
| 外科        | 条件调用   |
| 影像/ICU/内科 | 按需调用   |

👉 从“固定并行”升级为“智能调度”

---

### 2️⃣ 麻醉 Agent（已实现 ✅）

**职责：**

* 麻醉风险评估（ASA等）
* 麻醉方式与剂量建议
* 手术可行性评估（关键 gating）

**技术特点：**

* 基于**麻醉专属数据库（指南+规范）**
* RAG 检索增强

---

### 3️⃣ 外科 Agent

**职责：**

* 提出多种手术方案
* 评估手术收益 vs 风险
* 提供替代方案（保守治疗）

---

### 4️⃣ 专科扩展 Agent（可扩展）

例如：

* 影像科（诊断确认）
* ICU（重症评估）
* 内科（合并症管理）

👉 模块化扩展能力（系统亮点）

---

### ⭐ 5️⃣ 患者 Agent（现实建模）

**职责：**

* 模拟真实患者决策行为

**考虑因素：**

* 💰 经济能力
* 🧠 心理状态
* 👨‍👩‍👧 家庭支持
* ⚖️ 风险偏好

---

### ⭐ 6️⃣ Headmaster Agent（强化版）

**角色升级：**

> 不只是“选方案”，而是**主治医生级决策**

**职责：**

#### ① 综合决策

* 汇总 MDT + 患者信息
* 给出最终治疗方案

#### ② 手术建议

* 是否手术
* 手术方式选择

#### ③ 预后评估（新增 ⭐）

* 生存率 / 风险
* 并发症概率
* 康复周期

#### ④ 综合建议（新增 ⭐）

* 非手术路径
* 生活方式建议
* 随访方案

#### ⑤ 医患沟通视角（新增 ⭐）

* 如何向患者解释方案
* 风险沟通策略

---

### 7️⃣ 状态检测模块（Dynamic Loop）

**职责：**

* 持续监测患者状态变化
* 触发重新 MDT 评估

👉 实现“动态医疗决策系统”

---

## 🗄️ 数据体系（强化亮点 ⭐）

本系统采用 **“Agent专属数据库”架构**：

| Agent      | 数据来源         |
| ---------- | ------------ |
| 麻醉 Agent   | 麻醉指南 / 药物数据库 |
| 外科 Agent   | 手术指南 / 临床路径  |
| MDT Agent  | 综合临床规则       |
| Headmaster | 综合知识融合       |

**特点：**

* 📚 权威（临床指南/文献）
* 🔍 可检索（RAG）
* 🧩 解耦（每个Agent独立知识）

👉 避免“一锅模型”，提升可信度

---

## 🔄 决策机制（升级）

### ✅ 从：

**并行 → 汇总**

### 🚀 到：

**MDT调度 → 专科协作 → 主治决策 → 患者反馈 → 动态循环**
