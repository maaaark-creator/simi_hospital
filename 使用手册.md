# 使用手册

本文档用于说明本项目的运行方式、演示入口、配置模式以及配套展示病例。若需要了解项目背景、总体架构、展示逻辑和设计优势，请先阅读根目录 [README.md](/d:/hemu_STU/BME/simi_hospital/README.md:1)。

## 1. 快速使用

### 1.1 启动交互式页面

```bash
python mdt_chat_server.py
```

默认行为如下：

- 启动本地 HTTP 服务。
- 默认地址为 `http://127.0.0.1:8787/chat`。
- 自动打开浏览器页面。
- 前端展示病例输入、系统回复和实时执行进度。

如需禁止自动打开浏览器：

```bash
python mdt_chat_server.py --no-open
```

可选参数：

- `--host`：指定监听地址。
- `--port`：指定监听端口。

例如：

```bash
python mdt_chat_server.py --host 127.0.0.1 --port 8787 --no-open
```

### 1.2 运行完整演示脚本

```bash
python run_full_mdt_neurosurgery_demo_verbose.py
```

该脚本会：

- 使用根目录内置病例文本。
- 打印完整执行日志。
- 输出工作流结构化结果。
- 将结果保存为根目录 JSON 文件。

该入口适合用于检查完整链路是否正常，也适合做命令行演示。

### 1.3 运行总控流程

```bash
python head_doctor/run_head_doctor_demo.py
```

该脚本是根目录统一总控的命令行入口，支持读取 `multi_agent_config.json`，也支持通过命令行参数覆盖配置。

例如：

```bash
python head_doctor/run_head_doctor_demo.py --mode local --specialist-mode heuristic --mdt-mode heuristic --final-mode heuristic
```

若需要指定病例文件：

```bash
python head_doctor/run_head_doctor_demo.py --patient-file your_case.json
```

### 1.4 运行单专科演示

麻醉科：

```bash
python Anesthesia/run_anesthesia_pipeline_demo.py --mode local --decision-mode heuristic
```

心内科：

```bash
python Cardiology/run_cardiology_pipeline_demo.py --mode local --decision-mode heuristic
```

肝胆胰外科：

```bash
python "Hepatobiliary and Pancreatic Surgery/run_hepatobiliary_pipeline_demo.py" --mode local --decision-mode heuristic
```

神经外科：

```bash
python Neurosurgery/run_neurosurgery_pipeline_demo.py --mode local --retrieval-mode heuristic --decision-mode heuristic
```

骨科：

```bash
python Orthopaedics/run_orthopaedics_pipeline_demo.py --mode local --decision-mode heuristic
```

## 2. 运行模式说明

根目录通过 `multi_agent_config.py` 和 `multi_agent_config.json` 统一管理配置。当前系统采用“输入模式 + 专科模式 + 分诊模式 + 最终整合模式”的组合设计，以兼容本地演示和 API 增强两类场景。

### 2.1 输入模式

- `mode=local`
  - 适用于本地结构化病例输入。
  - 一般直接使用内置字典病例或 JSON 病例文件。
- `mode=api`
  - 适用于自然语言自由文本输入。
  - 系统会先调用模型将病例结构化。

### 2.2 专科决策模式

- `specialist_mode=heuristic`
  - 专科使用本地检索和本地决策逻辑。
  - 更稳定，便于流程验证。
- `specialist_mode=api`
  - 专科在检索和结论生成阶段引入模型能力。
  - 更适合展示语义理解和表达能力。

### 2.3 MDT 分诊模式

- `mdt_mode=heuristic`
  - `MDT_Call` 使用本地检索和规则生成首轮分诊及后续追问任务。
- `mdt_mode=api`
  - `MDT_Call` 使用模型生成分诊任务与追问规划。

### 2.4 最终整合模式

- `final_mode=heuristic`
  - `head_doctor` 使用本地汇总逻辑形成最终结构化输出。
- `final_mode=api`
  - `head_doctor` 使用模型对多专科结果进行综合整合。
  - 当前实现中还支持多模型审阅后再合成最终意见。

### 2.5 其它关键开关

- `use_mdt_for_uncertainty`
  - 控制 `head_doctor` 识别出的存疑点是否回流给 `MDT_Call` 继续追问。
- `use_api_for_clarification`
  - 控制专科对二次追问的补充说明是否调用 API。
- `patient_file`
  - 可指定外部病例文件路径。

### 2.6 API 凭据配置

如果需要启用真实 LLM 调用，必须在本机配置教学组分发的网关凭据。推荐在仓库根目录新建 `.env` 文件：

```env
HOSPITAL_LLM_GATEWAY_URL=http://hospital-llm-gateway:8080
HOSPITAL_LLM_API_KEY=这里填写教学组分发的真实key
```

注意：

- `.env` 只放在自己电脑上，不要提交到 Git。
- 仓库中的 `.env.example` 只作为模板，不包含真实 key。
- 代码只会在真正调用 API 模式时检查 `HOSPITAL_LLM_API_KEY`，启动本地演示不会因为没有 key 而失败。
- 如果没有 key，请使用 `heuristic` 本地模式，或在网页中关闭 `API分诊`、`API专科` 等 API 相关开关。

未配置 key 但启用 API 模式时，系统会提示：

```text
未配置 HOSPITAL_LLM_API_KEY，请配置后再启用 API 模式。如果只是本地演示，请关闭 API 模式。
```

## 3. 推荐演示方案

如果你的目标是课程展示或答辩，建议优先采用下面几种方案。

### 3.1 稳定展示方案

```bash
python head_doctor/run_head_doctor_demo.py --mode local --specialist-mode heuristic --mdt-mode heuristic --final-mode heuristic
```

适用场景：

- 需要保证现场运行稳定。
- 重点展示流程结构和模块组织。
- 不依赖外部模型服务。

### 3.2 强化展示方案

```bash
python mdt_chat_server.py
```

配合本手册后面的示例病例，在页面中直接输入病例文本进行演示。

适用场景：

- 需要展示交互页面和实时进度。
- 需要突出系统的多轮调度、专科协作和结构化输出。
- 适合答辩或项目路演。

### 3.3 多轮方案收束展示方案

```bash
python mdt_chat_server.py
```

建议使用同一患者分两轮输入：

- 第一轮输入主体病例。
- 第二轮补充关键检查或围术期条件。

适用场景：

- 需要展示 `memory`
- 需要展示多轮对话
- 需要展示终局方案如何从摘要收束为执行步骤

## 4. 展示病例

下面整理了几套适合直接用于展示的病例。建议按照“病例主体 + 补充信息”的方式演示，这样更容易体现系统的首轮分诊、二次追问、多轮更新和终局方案生成能力。

### 4.1 病例一：高龄股骨颈骨折合并房颤抗凝

适合展示：

- 骨科、心内科、麻醉科协同
- 围术期心血管风险与抗凝处理
- 多轮补充后方案是否能收束

第一轮输入：

```text
患者女，76岁。因“跌倒后左髋部疼痛、不能站立6小时”急诊入院。
既往有高血压15年、2型糖尿病10年、冠心病病史3年、房颤病史2年，长期口服缬沙坦、二甲双胍、阿司匹林及利伐沙班。
查体：左下肢短缩、外旋畸形，左髋部压痛明显，不能负重。血压168/96 mmHg，心率112次/分，律不齐，SpO2 97%。
化验：Hb 96 g/L，Cr 128 umol/L，Glu 11.2 mmol/L，INR 1.8，BNP轻度升高。
影像：X线提示左侧股骨颈骨折；胸片未见明确感染灶。心电图提示房颤伴快速心室率。
目前拟行髋部骨折手术治疗，但需评估抗凝药影响、围术期心血管风险、麻醉耐受性及手术时机。
```

第二轮补充信息：

```text
补充信息：患者末次服用利伐沙班时间距今已超过36小时，复查 INR 1.2，Hb 98 g/L；超声心动图提示左室射血分数58%，无明显新发室壁运动异常；麻醉评估认为可行全麻或椎管内麻醉，但建议先完成心率控制；骨科希望24小时内完成手术；目前 ICU 床位暂不需要，普通监护床位已协调。
```

### 4.3 病例三：急性 Stanford A 型主动脉夹层

适合展示：

- 急危重症场景
- 高风险围术期管理
- 心血管、麻醉、总控协同

第一轮输入：

```text
患者女，76岁。
因“突发胸背部撕裂样剧痛6小时，伴大汗、气促、烦躁不安”急诊入院。
患者6小时前在家中无明显诱因下突发胸前区及背部剧烈疼痛，疼痛呈撕裂样，向肩背部放射，伴明显胸闷、呼吸困难、恶心，无呕吐。家属诉患者随后出现短暂意识模糊，站立不稳，送医途中一度血压下降。
急诊CTA提示：Stanford A型主动脉夹层，自升主动脉至主动脉弓及降主动脉近段广泛受累，累及头臂干开口；心包少量积液；左侧胸腔少量积液。
床旁超声提示升主动脉增宽，少量心包积液，左室舒张功能减低。
目前考虑急性Stanford A型主动脉夹层，存在主动脉破裂及心包填塞风险，病情极危重。

既往有高血压15年，平素控制不佳；冠心病病史4年；慢性肾功能不全3年；2型糖尿病8年。
3年前曾因心房颤动长期服用利伐沙班，近半年服药不规律；平素口服厄贝沙坦、美托洛尔、阿托伐他汀、二甲双胍。
入院时生命体征：血压右上肢82/54 mmHg，左上肢106/68 mmHg，心率126次/分，呼吸28次/分，血氧饱和度93%，体温36.2摄氏度。
```

第二轮补充信息：

```text
补充信息：血库已备足红细胞、血浆和冷沉淀；体外循环团队已到位；床旁再次评估提示目前神志可配合但循环仍不稳定，已启动去甲肾上腺素维持灌注；术后 ICU 已确认收治；麻醉科建议立即行有创动脉压监测、中心静脉置管和必要时TEE监测。
```

适合展示：

- 肝胆胰外科主场景

- 规则模式和 API 模式都比较容易出结果

第一轮输入：

```text

拟急诊评估 ERCP 或其他胆道减压，禁食情况和气道评估暂不清楚。
```

第二轮补充信息：

```text
补充信息：经补液和抗感染后血压回升至102/64 mmHg；复查乳酸下降；麻醉科评估气道开放度尚可但建议在监护条件下实施；肝胆外科倾向先行 ERCP 减压；禁食已满8小时。
```

## 5. 各病例的推荐展示重点

- 病例一：适合强调骨科手术时机、抗凝处理和围术期心血管评估。
- 病例二：适合强调神经外科主导、麻醉协作和术中术后管理。
- 病例三：适合强调急危重症、多学科快速协同和高风险方案。
- 病例四：适合强调肝胆外科决策、感染控制和减压时机选择。

如果你想突出本次新改动，建议优先展示病例一或病例二，因为这两类病例更容易通过第二轮补充信息明显体现：

- 首轮先给出综合判断。
- 第二轮再依据补充结果收束为更明确的执行方案。

## 6. 推荐查看顺序

建议按以下顺序理解和使用项目：

1. 先阅读根目录 `README.md`，理解项目定位、结构图和展示病例。
2. 再阅读本手册，确认运行入口、模式设置和病例使用方式。
3. 先运行 `head_doctor/run_head_doctor_demo.py` 或完整 demo，验证后端流程。
4. 最后运行 `mdt_chat_server.py`，做交互式展示。

## 7. 注意事项

- API 模式依赖外部模型服务，运行前应确认网络和服务可用。
- 含空格的目录路径在命令行中需要加引号。
- 本项目当前为教学和研究原型，不用于真实临床决策。
- 如果要展示当前版本新增的多轮与终局方案能力，建议务必采用“同一病例分两轮输入”的方式，而不是一次性把所有信息都写满。
