# 使用手册

本文档用于说明本项目的运行方式、演示入口和配置模式。若需要了解项目背景、总体架构、展示病例和设计优势，请先阅读根目录 [README.md](/d:/hemu_STU/BME/simi_hospital/README.md:1)。

## 1. 快速使用

### 1.1 启动交互式页面

```bash
python mdt_chat_server.py
```

默认行为如下：

- 启动本地 HTTP 服务。
- 默认地址为 `http://127.0.0.1:8787/chat`。
- 自动打开浏览器页面。
- 前端展示病例输入、系统回复和实时执行进度。

如需禁止自动打开浏览器：

```bash
python mdt_chat_server.py --no-open
```

可选参数：

- `--host`：指定监听地址。
- `--port`：指定监听端口。

例如：

```bash
python mdt_chat_server.py --host 127.0.0.1 --port 8787 --no-open
```

### 1.2 运行完整演示脚本

```bash
python run_full_mdt_neurosurgery_demo_verbose.py
```

该脚本会：

- 使用根目录内置病例文本。
- 打印完整执行日志。
- 输出工作流结构化结果。
- 将结果保存为根目录 JSON 文件。

该入口适合用于检查完整链路是否正常，也适合做命令行演示。

### 1.3 运行总控流程

```bash
python head_doctor/run_head_doctor_demo.py
```

该脚本是根目录统一总控的命令行入口，支持读取 `multi_agent_config.json`，也支持通过命令行参数覆盖配置。

例如：

```bash
python head_doctor/run_head_doctor_demo.py --mode local --specialist-mode heuristic --mdt-mode heuristic --final-mode heuristic
```

若需要指定病例文件：

```bash
python head_doctor/run_head_doctor_demo.py --patient-file your_case.json
```

### 1.4 运行单专科演示

麻醉科：

```bash
python Anesthesia/run_anesthesia_pipeline_demo.py --mode local --decision-mode heuristic
```

心内科：

```bash
python Cardiology/run_cardiology_pipeline_demo.py --mode local --decision-mode heuristic
```

肝胆胰外科：

```bash
python "Hepatobiliary and Pancreatic Surgery/run_hepatobiliary_pipeline_demo.py" --mode local --decision-mode heuristic
```

神经外科：

```bash
python Neurosurgery/run_neurosurgery_pipeline_demo.py --mode local --retrieval-mode heuristic --decision-mode heuristic
```

骨科：

```bash
python Orthopaedics/run_orthopaedics_pipeline_demo.py --mode local --decision-mode heuristic
```

## 2. 运行模式说明

根目录通过 `multi_agent_config.py` 和 `multi_agent_config.json` 统一管理配置。当前系统采用“输入模式 + 专科模式 + 分诊模式 + 最终整合模式”的组合设计，以兼容本地演示和 API 增强两类场景。

### 2.1 输入模式

- `mode=local`
  - 适用于本地结构化病例输入。
  - 一般直接使用内置字典病例或 JSON 病例文件。
- `mode=api`
  - 适用于自然语言自由文本输入。
  - 系统会先调用模型将病例结构化。

### 2.2 专科决策模式

- `specialist_mode=heuristic`
  - 专科使用本地检索和本地决策逻辑。
  - 更稳定，便于流程验证。
- `specialist_mode=api`
  - 专科在检索和结论生成阶段引入模型能力。
  - 更适合展示语义理解和表达能力。

### 2.3 MDT 分诊模式

- `mdt_mode=heuristic`
  - `MDT_Call` 使用本地检索和规则生成首轮分诊及后续追问任务。
- `mdt_mode=api`
  - `MDT_Call` 使用模型生成分诊任务与追问规划。

### 2.4 最终整合模式

- `final_mode=heuristic`
  - `head_doctor` 使用本地汇总逻辑形成最终结构化输出。
- `final_mode=api`
  - `head_doctor` 使用模型对多专科结果进行综合整合。
  - 当前实现中还支持多模型审阅后再合成最终意见。

### 2.5 其它关键开关

- `use_mdt_for_uncertainty`
  - 控制 `head_doctor` 识别出的存疑点是否回流给 `MDT_Call` 继续追问。
- `use_api_for_clarification`
  - 控制专科对二次追问的补充说明是否调用 API。
- `patient_file`
  - 可指定外部病例文件路径。

## 3. 推荐演示方案

如果你的目标是课程展示或答辩，建议优先采用下面两种方案。

### 3.1 稳定展示方案

```bash
python head_doctor/run_head_doctor_demo.py --mode local --specialist-mode heuristic --mdt-mode heuristic --final-mode heuristic
```

适用场景：

- 需要保证现场运行稳定。
- 重点展示流程结构和模块组织。
- 不依赖外部模型服务。

### 3.2 强化展示方案

```bash
python mdt_chat_server.py
```

配合 README 中的示例病例，在页面中直接输入病例文本进行演示。

适用场景：

- 需要展示交互页面和实时进度。
- 需要突出系统的多轮调度、专科协作和结构化输出。
- 适合答辩或项目路演。

## 5. 推荐查看顺序

建议按以下顺序理解和使用项目：

1. 先阅读根目录 `README.md`，理解项目定位、结构图和展示病例。
2. 再阅读本手册，确认运行入口和模式设置。
3. 先运行 `head_doctor/run_head_doctor_demo.py` 或完整 demo，验证后端流程。
4. 最后运行 `mdt_chat_server.py`，做交互式展示。

## 6. 注意事项

- API 模式依赖外部模型服务，运行前应确认网络和服务可用。
- 含空格的目录路径在命令行中需要加引号。
- 本项目当前为教学和研究原型，不用于真实临床决策。
